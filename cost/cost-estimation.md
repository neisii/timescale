---

## 2. 전제 조건

- **실행 플로우**: `PLANNER(1회) → BUILDER(1회) → [REVIEWER → FIXER](×2) → REVIEWER(최종 1회)`
- **누적 컨텍스트**: 각 Agent는 이전 `conversation/` 문서를 모두 읽으므로 후반부로 갈수록 입력 토큰 증가
- **FIXER 모델 분배**: REVIEWER Category 기준 Cat.A 40%(Haiku 4.5) / Cat.B+C 60%(Sonnet 4.6)

---

## 3. 컴포넌트별 토큰 추정

### PLANNER (Claude Opus 4.7 — 1회)

Opus 4.7 신규 토크나이저로 인해 동일 텍스트 대비 +35% 오버헤드 적용.

| 항목 | 원본 추정 | +35% 보정 후 |
|---|---|---|
| 입력 (spec.md + AGENTS.md + role doc + 시스템) | ~8,000 | **~10,800** |
| 출력 (아키텍처 설계 전문 6개 섹션 + TODO) | ~10,000 | **~13,500** |

### BUILDER (Claude Sonnet 4.6 — 1회)

| 항목 | 토큰 | 내용 |
|---|---|---|
| 입력 | ~18,000 | P1 설계 문서(10K) + spec.md(3.5K) + role docs(4K) |
| 출력 | ~9,000 | 전체 코드베이스(~1,700 lines) + Notes |

### REVIEWER (Claude Sonnet 4.6 — 3회)

누적 컨텍스트로 인해 실행 횟수가 증가할수록 입력 토큰 증가.

| 실행 | 입력 토큰 | 출력 토큰 | 참조 문서 |
|---|---|---|---|
| Run 1 | ~27,000 | ~4,000 | P1 + P2 |
| Run 2 | ~34,000 | ~4,000 | P1 + P2 + P4_01 |
| Run 3 | ~50,000 | ~3,000 | P1 + P2 + P4_01 + P4_02 |
| **소계** | **~111,000** | **~11,000** | |

### FIXER (Haiku 4.5 / Sonnet 4.6 — 2회)

| 실행 | 전체 입력 | Haiku 4.5 분 | Sonnet 4.6 분 | 전체 출력 |
|---|---|---|---|---|
| Run 1 | ~30,000 | 12,000 (40%) | 18,000 (60%) | ~5,000 |
| Run 2 | ~40,000 | 16,000 (40%) | 24,000 (60%) | ~4,000 |
| **소계** | **~70,000** | **28,000** | **42,000** | **~9,000** |

## 5. 프롬프트 캐싱 적용 시 절감 효과

`conversation/` 파일은 REVIEWER·FIXER가 매 실행마다 반복 참조하므로 캐싱 효과가 크다.

- Sonnet 4.6 캐시 히트 단가: **$0.30/MTok** (기본 $3/MTok 대비 **10× 절감**)
- 추정 캐시 히트율: 60% (반복 참조되는 P1, P2 문서 기준)

| 구분 | 캐싱 없음 | 캐싱 적용 |
|---|---|---|
| REVIEWER ×3 입력 비용 | $0.333 | ~$0.107 |
| FIXER Sonnet 입력 비용 | $0.126 | ~$0.045 |
| **전체 예상 비용** | **~$1.33** | **~$0.88** |

## 7. Claude Pro 토큰 한도 기준

Claude Pro($20/월)는 공식 월간 토큰 수치를 공개하지 않으나, API 단가 역산으로 소비 비중 추정.

| 항목 | 값 |
|---|---|
| Claude Pro 월 요금 | $20 / 월 |
| 이 프로젝트 API 환산 비용 (캐싱 미적용) | ~$1.33 |
| 이 프로젝트 API 환산 비용 (캐싱 적용) | ~$0.88 |
| **월 예산 대비 소비 비중** | **약 4~7%** |

즉, Claude Pro 한 달 사용량의 약 **1/15 ~ 1/25 수준**으로 이 MVP 구현 1회 완전 실행이 가능하다.

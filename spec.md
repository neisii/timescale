⸻

0. 목표 정의

당신은 산업용 시계열 데이터 처리 시스템을 설계/구현하는 백엔드 엔지니어입니다.
다음 요구사항을 기반으로 현실적인 데이터 흐름과 운영 시나리오를 고려한 MVP 수준의 시스템을 설계하고 구현하세요.

이 프로젝트의 핵심 목표는 다음과 같습니다:

* 시계열 데이터 처리 구조 이해
* 이벤트 기반 데이터 파이프라인 설계
* 데이터 정합성 유지 전략 구현
* 장애 상황 대응 설계
* 운영 가능한 시스템 설계 능력 증명

AI 모델 구현은 제외하며, 데이터 흐름과 시스템 설계에 집중합니다.

⸻

1. 전체 아키텍처 설계

다음 구조를 기반으로 시스템을 설계하세요:

Producer → Kafka → Consumer → DB → API

각 컴포넌트 역할 정의

1. Producer

* IoT 센서 시뮬레이터 역할
* 시간 기반 시계열 데이터 생성
* 정상/이상/결측/지연 데이터 포함

2. Kafka

* 이벤트 스트림 브로커
* 파티셔닝 기반 처리 분산
* 메시지 순서 및 지연 관리

3. Consumer

* 이벤트 소비 및 처리
* 데이터 정제 및 검증
* 중복 제거 및 시간 기준 필터링 수행

4. Database

* 시계열 데이터 저장
* PostgreSQL 또는 TimescaleDB 사용

5. API (FastAPI)

* 설비 상태 조회
* 이상 이벤트 조회
* 시간 범위 기반 조회

⸻

데이터 시나리오 요구사항

다음 상황을 반드시 포함해야 한다:

1. 정상 데이터

* 일정 범위 내 값 유지

2. 이상 데이터

* 급격한 spike 또는 drop

3. 결측 데이터

* 특정 구간 null 또는 누락

4. 노이즈

* 랜덤 변동 포함

5. 지연 데이터 (out-of-order)

* 과거 timestamp 데이터가 늦게 도착
